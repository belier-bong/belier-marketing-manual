📋시스템 개요

이 시스템은 무엇인가요?

구글 시트 "2. 생산 현황"에 연결된 Apps Script입니다. 각 행(=스타일)마다 작업지시서(작지) 스프레드시트 URL을 연결하면, 생산정보/원단일정/자재현황을 자동으로 불러오고, 현재 공정 상태와 원단 상태를 자동 계산합니다.

시스템은 크게 두 종류의 갱신을 합니다:

빠른 갱신 (시트 내 계산)

이미 시트에 있는 데이터를 기반으로 상태값만 다시 계산합니다. 외부 연동 없이 즉시 완료됩니다.

데이터 갱신 (작지 연동)

각 행의 작지 URL을 열어서 최신 데이터를 다시 불러옵니다. 시간이 오래 걸리며 5분 제한이 있습니다.

⚡빠른 갱신 (시트 내 계산)

빠른 갱신이란?

이미 시트에 입력된 데이터만으로 상태값을 다시 계산하는 기능입니다. 외부 작지를 열지 않기 때문에 수 초 만에 전체 행이 갱신됩니다.

🚀 생산 관리 > ⚡ 빠른 갱신
🔄 현재공정 (AE열) — 체크박스 기반 공정 상태 재계산
📅 원단상태 (AF열) — BF/BG 날짜 기반 상태 재계산
🧵 원단일정 보정 (BF/BG열) — MM.DD → YY.MM.DD 자동 변환
📊 자재요약 (AG/AH열) — BJ~BQ 데이터 기반 요약 재계산
🚫 후가공 처리 (AR/AS열) — 후가공 "진행안함" 행 자동 비활성화
🎨 체크박스 색상 (AL~AW열) — 체크된 셀 배경색 일괄 적용
───
✨ 전체 빠른 갱신 — 위 6가지를 순서대로 모두 실행
추천 사용법
데이터를 직접 수정하거나, 체크박스를 일괄 변경한 후에 "✨ 전체 빠른 갱신"을 실행하면 모든 상태값이 일괄로 맞춰집니다.
🔄 현재공정 (AE열)

체크박스(AL~AW) 상태와 원단 상태(AF)를 보고 현재 공정 단계를 자동으로 판정합니다.

갱신 열: AE

참조 열: AF, AI, AJ, AL~AW

📅 원단상태 (AF열)

발주일(BF)과 입고일(BG)을 오늘 날짜와 비교해서 원단 상태를 판정합니다.

갱신 열: AF

참조 열: BF, BG

🧵 원단일정 보정 (BF/BG열)

BF/BG 열의 날짜를 YY.MM.DD 형식으로 통일합니다. MM.DD만 입력된 경우 연도를 자동 추정합니다 (미래 6개월 이상이면 작년으로).

📊 자재요약 (AG/AH열)

BJ(발주율)과 BL(투입율)을 조합해서 AG열에 "80/60%" 형태로 요약합니다. 공용부자재도 동일하게 AH열에 표시합니다.

🏭생산정보 갱신 (Y~AD열)

각 행의 작지 URL(I열)을 열어서 패턴업체, 생산처, 생산국, 작지일, 생산구분, 후가공 유형을 다시 불러옵니다.

🚀 생산 관리 > 🏭 생산정보 갱신 (Y~AD열)
▶ 전체 행 (자동 이어하기) — 6행부터 끝까지 순차 실행
6~55행 / 56~105행 / ... / 356~405행 (50행 단위)
갱신 열: Z(패턴업체), AA(생산처), AB(생산국), AC(작지일), AD(생산구분), AI(후가공)
소요시간: 50행당 약 2~4분 (작지 수에 따라 다름)
자동 이어하기: 5분 초과 시 자동 중단 → 다시 실행하면 중단 지점부터 재개
5분 제한
구글 앱스크립트는 한 번 실행에 최대 5분까지만 돌아갑니다. "전체 행 (자동 이어하기)"를 사용하면 5분 지나면 자동으로 멈추고, 다시 실행하면 멈춘 곳부터 이어서 진행합니다.

🧵원단일정 갱신 (BF~BG열)

각 행의 작지에서 원단 발주일(BF)과 입고일(BG)을 다시 불러옵니다. 연동 후 AF(원단상태)와 AE(현재공정)도 자동으로 재계산됩니다.

🚀 생산 관리 > 🧵 원단일정 갱신 (BF~BG열)
▶ 전체 행 (자동 이어하기)
6~55행 / 56~105행 / ... / 356~405행
갱신 열: BF(발주일), BG(입고일)
연쇄 갱신: AF(원단상태), AE(현재공정) 자동 재계산
날짜 형식: YY.MM.DD (예: 25.03.15)
작지에 날짜가 수식으로 연결된 경우
작지의 발주일/입고일이 ='자재발주'!E13 같은 수식으로 되어있으면, 초기 연동 시점에 자재발주 시트에 날짜가 없었다면 빈 값으로 불러옵니다. 나중에 날짜가 채워져도 자동으로 갱신되지 않으므로 이 메뉴에서 수동 갱신하거나, v5.40의 스마트 갱신을 사용하세요.

📦자재현황 갱신 (BJ~BQ열)

각 행의 작지에서 "자재발주" 시트를 열어, 해당 품번의 부자재 발주/투입 현황을 계산합니다.

🚀 생산 관리 > 📦 자재현황 갱신 (BJ~BQ열)
▶ 전체 행 (자동 이어하기)
6~55행 / 56~105행 / ... / 356~405행
갱신 열: BJ~BQ (발주율, 미발주목록, 투입율, 미투입목록 × 부자재/공용)
자재 구분: "부자재" → BJ~BM, "공용부자재" → BN~BQ ("원자재"는 제외)

📋전체 데이터 갱신

생산정보 + 원단일정 + 자재현황을 한 번에 전부 불러옵니다. 가장 시간이 오래 걸리지만 가장 완전한 갱신입니다.

🚀 생산 관리
📋 전체 데이터 갱신 (50행) — 50행 단위 선택 실행
📋 전체 데이터 갱신 (자동 이어하기) — 6행부터 끝까지 자동
갱신 범위: Y~AD + BF~BG + BJ~BQ + AE + AF + AR/AS (전부)
소요시간: 전체 400행 기준 약 20~40분 (5분 × 여러 번 반복)
주의: 시간이 매우 오래 걸립니다
모든 행의 작지를 하나씩 열어서 데이터를 가져오기 때문에, 전체 갱신은 5분 제한으로 여러 번 반복해야 합니다. 꼭 전체를 돌려야 하는 경우가 아니라면 v5.40 스마트 갱신이나 부분 갱신(50행 단위)을 권장합니다.

🆕스마트 갱신 (v5.40 신규)

v5.40에서 새로 추가된 기능

기존 원단일정 갱신은 모든 행을 돌렸기 때문에 이미 날짜가 있는 행도 다시 불러와서 시간이 낭비되었습니다. 스마트 갱신은 BF(발주일)이나 BG(입고일)이 비어있는 행만 골라서 작지에서 불러옵니다.

🚀 생산 관리 > 🆕 스마트 갱신
🧵 빈 날짜만 갱신 (BF/BG 비어있는 행)
🔄 빈 날짜 + 발주전 행 갱신
───
⏰ 매일 7시 빈 날짜 자동갱신 설정
⏰ 빈 날짜 자동갱신 해제
🧵 빈 날짜만 갱신

BF 또는 BG가 비어있고, 작지 URL이 있는 행만 찾아서 날짜를 불러옵니다.

언제 쓰나: 새 스타일을 추가했거나, 작지에 날짜가 나중에 채워진 경우

결과: 완료 후 성공/실패 리포트가 팝업으로 뜹니다

핵심 장점
이미 날짜가 있는 행은 건너뛰므로, 기존 전체 갱신보다 훨씬 빠릅니다.
🔄 빈 날짜 + 발주전 행 갱신

빈 날짜 행 + AF가 "발주전"인 행까지 포함해서 재시도합니다.

언제 쓰나: 이전에 발주일이 없어서 "발주전"이었는데, 이후 작지에 발주일이 입력된 경우

추가 대상: AF열이 "발주전"인 행 (URL이 있는데 날짜만 없는 상태)

해제: "⏰ 빈 날짜 자동갱신 해제" 클릭

기존 6시 트리거와의 관계
기존 6시 트리거는 시트 내 데이터만 재계산(빠른 갱신)합니다. 7시 트리거는 실제로 작지를 열어서 빈 날짜를 채워주므로, 6시 갱신 후에 실행되도록 1시간 간격을 둔 것입니다.

셀을 편집하면 자동으로 실행되는 기능들입니다. 별도 메뉴 실행 없이 바로 동작합니다.

→ AE(현재공정), AF(원단상태) 자동 계산, AR/AS 후가공 처리

체크박스 날짜 일괄 입력
26년 12월 → 25년 12월 변환

BF/BG열에서 "26.12.XX"로 잘못 입력된 날짜를 "25.12.XX"로 일괄 변환합니다.

배경: 12월 날짜가 오타로 26년으로 입력되면 D-287 같은 비정상 상태가 표시됩니다. 이 기능으로 한 번에 고칠 수 있습니다.

날짜 형식 변환

BF/BG열의 MM.DD 형식을 YY.MM.DD로 자동 변환합니다. 미래 6개월 이상이면 작년으로 처리합니다.

예시: 12.29 → 25.12.29 (오늘이 2026년 3월이면 작년인 25년으로)

체크박스 날짜 일괄 입력

체크는 되어있는데 옆 날짜열이 비어있는 경우, 오늘 날짜를 일괄 입력합니다.

예를 들어 AL(투입)이 체크인데 AM(투입일)이 비어있으면, AM에 오늘 날짜를 넣어줍니다.

🛠️문제 해결

자주 발생하는 문제와 해결법
Q. 원단상태가 D-287 같은 이상한 숫자로 표시돼요

BF 또는 BG에 26.12.XX 처럼 연도가 잘못 입력되었을 가능성이 높습니다. 📝 일괄 처리 > "26년 12월 → 25년 12월 변환"을 실행하거나, 해당 셀을 직접 수정하세요.

Q. 작지에 날짜가 있는데 생산현황에 안 불러와져요

처음 URL을 연결할 때 작지에 날짜가 없었고, 이후에 날짜가 추가된 경우입니다. 🆕 스마트 갱신 > "빈 날짜만 갱신"을 실행하면 빈 행만 다시 불러옵니다.

Q. 전체 갱신이 중간에 멈춰요

5분 제한 때문에 정상적으로 중단된 것입니다. 같은 메뉴를 다시 실행하면 중단된 곳부터 이어서 진행합니다. 완전히 처음부터 다시 하고 싶으면 설정 > "모든 진행상태 초기화"를 먼저 실행하세요.

Q. 메뉴가 안 보여요

시트를 새로고침(F5)하면 "🚀 생산 관리" 메뉴가 상단에 나타납니다. 처음 사용 시 권한 승인 팝업이 뜰 수 있는데, "허용"을 눌러주세요.

Q. "자동 이어하기"를 쓰는데 진행이 안 돼요

이전에 저장된 마지막 행이 실제 데이터 범위를 넘은 경우입니다. 설정 > "모든 진행상태 초기화"를 실행한 뒤 다시 시도하세요.

Q. 스마트 갱신 함수를 찾을 수 없다는 오류가 떠요

v5.40 추가 코드가 앱스크립트에 제대로 붙여넣어지지 않은 상태입니다. Apps Script 편집기에서 Ctrl+End로 맨 아래로 가서 function refreshEmptyFabricDatesOnly()가 있는지 확인하세요. 없으면 v5.40 코드를 기존 코드 아래에 붙여넣고 저장하세요.
